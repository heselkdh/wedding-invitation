# QR Code Generation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let the admin generate a print-ready PNG QR code (encoding the fixed invitation site URL) from a single button in the admin panel.

**Architecture:** Extract the site URL into a small shared constant module so `main.js` (Kakao share) and `admin.js` (QR code) don't each hardcode their own copy. Import the `qrcode` library directly by URL inside `admin.js` (it's already an ES module, same as its Firebase imports — `qrcode` has no prebuilt global `<script>` bundle, so a jsdelivr `+esm` import is the right fit rather than a classic `<script>` tag), and generate the QR code into a `<canvas>` on demand, entirely client-side — no Firestore/Cloudinary involvement.

**Tech Stack:** Vanilla JS (ES modules), `qrcode` (soldair/node-qrcode) UMD build via unpkg CDN. No test runner exists in this codebase — verification steps are manual browser checks via a local static server, matching the pattern used in the Fix & Polish plan.

### Task 3: Wire up QR code generation in admin.js

**Files:**
- Modify: `js/admin.js:1-11`

- [ ] **Step 1: Import the shared URL constant and the QR library**

`qrcode` has no prebuilt global `<script>` bundle (checked on unpkg — 404; its `package.json` only maps a `browser` field to an ES module, no bundled global). `admin.js` is already an ES module, so import the library the same way Firebase is imported — by URL, via jsdelivr's `+esm` endpoint (confirmed working: bundles the package's browser entry + its one dependency, `dijkstrajs`, into a single importable module).

Add to the top imports in `js/admin.js`:

```js
import { SITE_URL } from './site-config.js';
import QRCode from 'https://cdn.jsdelivr.net/npm/qrcode@1.5.4/+esm';
```

- [ ] **Step 2: Add the click handler**

Add this near the other panel-1 handlers in `js/admin.js` (e.g. right after the `save-info-btn` click handler):

```js
document.getElementById('qr-generate-btn').addEventListener('click', () => {
  const canvas = document.getElementById('qr-canvas');
  QRCode.toCanvas(canvas, SITE_URL, { width: 600, margin: 2 }, err => {
    if (err) {
      showToast(`QR 코드 생성 실패: ${err.message}`);
      return;
    }
    document.getElementById('qr-preview-wrap').style.display = 'block';
    document.getElementById('qr-download-link').href = canvas.toDataURL('image/png');
  });
});
```

- [ ] **Step 3: Manually verify (spy on the call to confirm correct content, then check the visual result)**

Serve the site, open `http://localhost:5173/admin.html`, log in, open "✏️ 기본 정보". In devtools console, import the module directly and wrap `toCanvas` to capture what it's called with, then click the button:

```js
const mod = await import('https://cdn.jsdelivr.net/npm/qrcode@1.5.4/+esm');
const original = mod.default.toCanvas;
let capturedText = null;
mod.default.toCanvas = (canvas, text, opts, cb) => {
  capturedText = text;
  return original(canvas, text, opts, cb);
};
document.getElementById('qr-generate-btn').click();
await new Promise(r => setTimeout(r, 300));
({
  capturedText,
  matchesSiteUrl: capturedText === 'https://heselkdh.github.io/wedding-invitation/',
  previewVisible: document.getElementById('qr-preview-wrap').style.display,
  downloadHref: document.getElementById('qr-download-link').href.slice(0, 30)
})
```

Note: this spy patches the module namespace object retrieved via a fresh `import()` in the console, which is the same cached module instance `admin.js` imported (the browser's module cache is keyed by URL) — so patching `mod.default.toCanvas` here does affect the call `admin.js`'s handler makes.

Expected: `matchesSiteUrl: true`, `previewVisible: "block"`, `downloadHref` starting with `"data:image/png;base64,"`.

Then visually confirm the canvas shows a scannable-looking QR pattern (not a blank square), and click "💾 PNG 다운로드" to confirm it downloads a `wedding-qr.png` file. Optionally scan it with a phone camera to confirm it opens the invitation site.
